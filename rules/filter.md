# 基金筛选卡 · 把模糊需求翻译成筛选条件

## 目的
用户用自然语言提需求，要准确翻译成结构化筛选条件，不多加、不漏掉。这是规划环节的核心。

## 可用的筛选维度（不在用户话里出现的，一律留空，不要脑补）
- region 区域：美国 / 欧洲 / 日本 / 印度 / 港股 / 亚太新兴 / 全球
- theme 主题：美国宽基 / 科技成长 / 半导体 / 医疗健康 / 消费 / 商品资源 / 债券收益 / 红利低波 / 综合配置
- role 用途：底仓候选 / 卫星配置 / 进攻仓
- fundType 类型：指数联接 / 主动基金 / 股票主动 / 混合主动 / 债券收益
- risk 风险：中 / 中高 / 高
- 收益门槛：近1年 / 近3月 / 今年以来 的最低百分比
- discountFee 申购费折后上限、成立年限上下限、晨星评级下限
- purchaseLimitYuanMin 申购可达性过滤：
  - "不限购" → 99999（只要开放申购）
  - "不限购或限购至少100/日" → 100
  - "能申购的（排除暂停）" → 0
  - 不提就留 null
- 排序字段 + 升降序 + 返回数量（默认综合评分降序，取 8 只）

## 行业词的正确归类（容易错的）
- "存储 / 储存 / 内存 / 半导体 / 芯片" → theme 填**半导体**，不要只填科技成长
- 中文语境"储存"通常指**存储芯片**，不是仓储地产、不是数据中心 REIT
- "纳指 / 纳斯达克 / 标普 / 美股大盘" → 偏**美国宽基**
- "中概 / 恒生科技 / 港股互联网" → region 港股 + theme 科技成长
- "黄金 / 原油 / 大宗" → theme 商品资源

## "最近很火 + 找几只" 必须当筛选，不是问行情
"最近 AI 很火，给我推荐几只相关 QDII" → 这是**筛选需求**，照样翻译成 theme=科技成长 给基金列表，不要回避说"暂未接入实时行情"。

## 软约束：用户画像（仅筛选场景、仅用户没明说时）
- 保守 → risk 限"中"，按夏普比率高到低排
- 中等 → risk 放"中/中高"，保持综合评分排序
- 激进 → 不加 risk 限制
- 长期持有倾向 → 成立年限至少 3 年
- 用户本轮明说的条件，永远盖过画像

## 给结果时
- 点名基金用「`代码 名称`」格式提一次，前端会渲染成卡片
- 简短说一句为什么是这几只（贵在便宜/评级高/近1年强），别干甩列表
- 如果筛出来很少或为空，说清楚是条件太严，并给放宽建议
- 绝不凭空补用户没提的条件（画像例外，且要遵守上面规则）
