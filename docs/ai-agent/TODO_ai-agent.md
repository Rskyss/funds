## TODO-7：会话历史列表（像 ChatGPT 那种） ✅ 已实现（2026-05-15）

**功能**
- 聊天面板顶部加"历史"按钮，点击展开会话列表
- 每条列表项：标题（第一条用户消息前 30 字）+ 相对时间（刚刚 / X 分钟前 / X 小时前 / X 天前 / 绝对日期）+ 消息条数
- 点击列表项 → 切换到该会话，自动拉回完整对话和卡片
- 每条右侧有 × 可删除
- "返回当前对话"按钮回到当前正在聊的会话

**实现路径**
- 后端 `GET /api/chat/sessions`：登录用户查 `chat_sessions` 表按 `updated_at desc`，返回标题、时间、计数
- 前端 `localStorage[qdii-compass-chat-sessions-index]` 维护本地索引（每发一条消息就 upsert）
- 打开列表时：登录态会拉后端会话列表与本地合并，去重 by sessionId
- 匿名用户：仅本地索引，跨浏览器看不到

## TODO-1：刷新页面后聊天抽屉的对话历史消失 ✅ 已修复（2026-05-15）

**已实施：方案 C（localStorage 优先 + 后端回退）+ 卡片持久化**

- 浏览器本地存最近 20 轮完整对话（消息文字 + 卡片 + 来源 + 意图标签）→ 刷新秒开
- 后端 `chat_sessions.state.messages` 现在每轮也存了卡片/来源/意图；`GET /api/chat/history?sessionId=xxx` 返回完整数据
- 换浏览器 / 清缓存 / 跨设备 都能恢复**完整对话**（包括卡片），不再只是文字
- "新会话"按钮一次性清掉两端缓存
- 已验证：新对话发送后查 history 能拿回 4 张卡片 + intent=filter 标签

**已知限制**：TODO-1 修复**之前**就在数据库里的旧会话，后端没存过卡片，那部分历史只能恢复出文字气泡，无法补救卡片。


### 现象
- 在"AI 投顾"抽屉里聊了几轮，左侧/右侧的对话气泡 + 基金卡片都出现了。
- 浏览器按 F5 或 Cmd+R 刷新页面后，再点开"AI 投顾"按钮，抽屉里只看到欢迎语，**之前的所有对话和卡片都消失了**。
- 但底部输入框继续问的话，AI 还能记住上下文（因为后端会话还在）。所以是"前端展示丢了"，不是"后端忘了"。

### 截图
2026-05-15 14:14 用户反馈截图保存在：`/Users/rs/.cursor/projects/Users-rs-Desktop-project/assets/image-f39e4c7d-c6ec-4846-9fa3-2fadccdc0a79.png`

### 根因（产品语言）
- 之前的对话只存在浏览器内存里，没存到浏览器本地存储。
- 后端 `chat_sessions` 表其实把所有消息都记下来了，但页面打开时没有主动去拉回来。
- 所以刷新一次，前端的"记忆"就清零了，体验上感觉对话凭空消失。

### 影响
- 用户体验下降：每次刷新都得重新提问、重新看到卡片。
- 用户可能误以为系统"忘事"，对 AI 的信任打折扣。
- 不影响后端数据完整性，也不影响付费/合规。

### 推荐修复方案（按工作量从小到大）

**方案 A：纯前端 localStorage 缓存（半天）**
- 每轮聊天结束后，把消息气泡 HTML、卡片 JSON、sources 一起存到 localStorage。
- 打开抽屉时优先用 localStorage 恢复。
- 优点：改动小、零后端开销。
- 缺点：换浏览器/换设备打不开历史；localStorage 容量上限 5MB，长期会有溢出风险。

**方案 B：后端拉历史接口（1 天）**
- 加 `GET /api/chat/history?sessionId=xxx` 接口，返回 `chat_sessions.state.messages` 与最近一轮的 cards。
- 前端打开抽屉时，根据 localStorage 里的 sessionId 调一次，把对话流恢复出来。
- 优点：跨浏览器一致，与服务端真相一致，登录后还能跟用户绑定。
- 缺点：每次打开抽屉多一次网络请求；要把 cards/sources 也存到 `chat_sessions.state` 里（目前只存了 messages）。

**方案 C（推荐）：A + B 结合**
- 默认走 localStorage，秒开。
- 没有 localStorage 时（换设备、清缓存）回退到后端拉。
- 登录用户切到新设备时也能找回对话。

### 决策需要的输入
- 是否上 B/C？需要后端服务多一个接口和一次拉取，是否接受额外延迟？
- 历史保留时长？建议 30 天，超过的自动清除（前后端都要清）。
- 未登录用户的 sessionId 是否要做"匿名→登录态绑定"？目前匿名用户的 sessionId 存浏览器，登录后并不会合并到 user_id 名下。

## TODO-3：基金数据缓存与 AI 路径的同步问题

### 现象
- `loadFundsCached` 在 `lib/agent/tools.mjs` 里有 60s TTL。
- 如果用户先点了"刷新数据"按钮，60s 内问 AI，AI 拿到的是旧数据。

### 处理建议
- 在 `server.mjs` 的 `refreshFunds()` 末尾调用 `invalidateFundsCache()` 把 AI 那侧的缓存也作废一下。
- 工作量：5 分钟。
