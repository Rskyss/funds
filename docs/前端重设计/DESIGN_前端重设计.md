# 前端重设计 · 设计文档（DESIGN）

> 依据 [ALIGNMENT_前端重设计.md](ALIGNMENT_前端重设计.md) 的决策。本文聚焦"怎么做"的落地细节。

## 一、整体策略

渲染层与数据/接口层解耦：现有 `app.js` / `chat.js` / `auth.js` 里的**数据获取、状态、API 调用逻辑保留**，只重写"生成 DOM/样式"的部分。`styles.css` 按原型 `compass.css` 的设计系统重建。原生 JS，零构建。

## 二、设计系统映射（第 0 期）

把原型 `fund/compass.css` 的 token 接入线上 `public/styles.css`，统一观感：

| 维度 | 现状 | 新值（采自原型） |
|---|---|---|
| 背景 | `#f8fafc` | `--bg:#F4F6FB` / `--bg-deep:#ECEFF7` |
| 主色 | `--blue:#2563eb` | `--accent:#1E40FF`（+ hover/soft/edge/glow 派生） |
| 涨/跌 | `#e11d48 / #059669` | `--up:#E0341F` / `--down:#0A8F5C`（中国习惯红涨绿跌，保持） |
| 文字分级 | text/muted/ink 3 级 | ink/ink-2/ink-3/muted/muted-2 5 级 |
| 圆角 | 16/10 | 14/10/6/pill 四档 |
| 阴影 | sm/md/lg | s/m/l（更克制，带主色辉光） |
| 字体 | 系统字体 | Inter + JetBrains Mono（数字等宽对齐，CDN 引入） |
| 缓动 | 杂 | 统一 `--easing` / `--easing-out` |

> 字体走 Google Fonts CDN（与原型一致）；若考虑离线/网络，可后续改本地托管，本期先对齐视觉。

## 三、后端精简字段契约（第 2 期，决策 1）

列表接口（现有返回 fund 数组的那个）每只基金**新增**：

```
sparkPoints: number[]   // 近1年约30个等距采样净值点；无数据则 []
top3: [{ code, name, ratio }]  // 前3大重仓；无 fund_details 则 []
```

- 来源：`nav_history` 降采样 + `fund_details` 持仓前三。
- 约束：单独计算，不改 `rowToFund` 主映射的现有字段；列表查询批量取，避免 N+1。
- 前端兜底：`sparkPoints` 空 → 卡片不画迷你图（留白或占位），不报错；`top3` 空 → 不显示 TOP3 行。
- 性能：30 点 × 数百只 ≈ 可接受；若实测偏重，降到 20 点或按需分页加载。**实测后定**。

## 五、每期验收标准

- **第 0 期**：页面整体配色/字体/圆角/阴影切到新系统，无功能回归，登录→列表→详情→AI 路径可走通。
- **第 1 期**：顶栏 + Hero + KPI 视觉还原原型；数字滚动动效；登录态正确；移动端不破版。
- **第 2 期**：卡片视觉还原；真实数据 + 真实迷你图/TOP3；筛选/排序/密度切换正常；错峰动效。
- **第 3 期**：详情抽屉所有区块还原且数据真实；经理面板、费率 tab、ESC 关闭等交互保留。
- **第 4 期**：AI 投顾外观还原；真实多轮对话/流式/历史/嵌入卡正常。
- **第 5 期**：登录/注册/对比页风格统一。
- **第 6 期**：关键路径回归全过，响应式自查通过，交付 FINAL + TODO。

## 六、回归走查清单（每期必走）

登录 → 列表加载 → 搜索 → 快捷筛选 → 排序 → 密度切换 → 打开详情 → 经理面板 → AI 点评 → 加入对比 → 对比页 → AI 投顾多轮对话 → 退出登录。
