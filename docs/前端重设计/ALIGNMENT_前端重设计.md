## 五、需要你拍板的设计决策（3 项）

> 这几项无法替你猜，影响体验与工作量，做之前要你定。

**决策 1：卡片上的"迷你走势图"和"TOP3 重仓股"从哪来？**
原型每张卡都有一条小走势图 + TOP3 持仓。线上现状：净值历史和持仓只在**点开详情时**才加载，列表接口里没有。
- 选项 A：卡片只显示真实收益数字，不放迷你图/TOP3（最快、零后端改动）
- 选项 B：迷你图用列表已有的收益数据"示意性"绘制，标注非真实净值（折中）
- 选项 C：后端列表接口补上每只基金的精简净值点 + TOP3（最还原，但要动后端，违背"后端不动"边界）

**决策 2：卡片密度默认值**
原型默认"舒适"（一屏看到的基金比现在少）。你的重度用户习惯一屏扫几十只。
- 提供"舒适/紧凑"切换并记住选择，默认用哪个？

**决策 3：登录入口怎么融进新顶栏**
原型顶栏右上是"AI 投顾 + 头像"，没有登录概念。线上是必须登录的。
- 登录后头像菜单（推荐）/ 顶部常驻登录按钮 / 进站先登录页

### 决策结论（已确认 2026-05-17）

- **决策 1 = 选项 C**：后端补真实数据，卡片 100% 还原原型。落地方式：列表接口新增轻量字段——每只基金近 1 年约 30 个采样净值点 + TOP3 重仓，单独精简返回，不影响现有详情接口；列表渲染做空值兜底（无净值点时卡片不画图，不报错）。
- **决策 2 = 默认「舒适」**：与原型一致；提供"舒适/紧凑"切换并本地记住用户选择。
- **决策 3 = 头像菜单**：登录后右上头像点出菜单（账号/退出）；未登录时该位置为「登录」按钮。

## 六、动效取舍清单（逐条可勾）

| 动效 | 处理 | 理由 |
|---|---|---|
| 大数字滚动进场（CountUp） | ✅ 保留 | 真·提质感，数据真实 |
| 走势图自绘展开 | ✅ 保留 | 高级感 |
| 卡片错峰滑入 | ✅ 保留 | 列表不生硬 |
| 抽屉/弹层统一缓动 | ✅ 保留 | 整体一致性 |
| 持仓/配置条形增长动画 | ✅ 保留 | 真实数据 |
| 状态徽标呼吸（暂停申购等） | ✅ 保留 | 注意力引导，真实状态 |
| 数字"实时闪烁红绿"（tick flash） | ⚠️ 改造 | 改为**仅在数据真刷新时**闪，不定时假闪 |
| 顶部行情滚动条（MarketTape） | ❌ 去掉 | 无真实数据源，纯装饰会误导 |
| 背景脉冲点（hero pulses） | ✅ 保留（默认开，可关） | 纯氛围，不涉及数据真实性 |

## 八、风险点

1. **回归风险**：线上业务逻辑多（2800+ 行），重画渲染层时可能误伤已跑通功能 → 对策：渲染层与数据/接口层解耦，分期小步、每期回归。
2. **无测试套件**：项目无自动化测试 → 对策：每期手动走查关键路径（登录→列表→筛选→详情→AI→对比）。
3. **真实数据形态漂移**：东方财富接口格式变过几次 → 不在本次改动范围，但重画时不假设数据一定齐全，做空值兜底。
4. **响应式**：原型按桌面宽屏设计，移动端需补断点。
